i-Tracker Event's Documentation

This document explains the different events the application listens and emits

Implement the events based on the description and sample examples

SocketUrl   api.i-Tracker.in/notify

Events To Listen

authenticate event

This event "auth" has to be listened on the client's end to verify user authentication.

Authentication status event

This event "authStatus" is listened by client for the authentication status
Issue Update Notification

This event "issue-updates-server" is listened by client to upon any update in issue's fields and notifies.
members of watchlist about the change.
Payload
Eg: 
                {
                  "issueId" : "Xiiu_6767",
                  "field" : "title",
                  "watchlist" : ['asfd545','UytC5656'],
                  "userId" : "ghfdgYT67"
                  
                }

Events To Emit

Authentication Handshake event

This event "auth" has to be emitted when user listens to authenticate event
Auth token is sent as payload in this event.
User will then recieve notification about auth status.
Issue Update event

This event "issue-updates-client" has to be emitted when user updates any issue's field.
The watchlist will be notified of this updates.
Payload Eg: 
            {
                "issueId" : "Xiiu_6767",
                "field" : "title",
                "watchlist" : ['asfd545','UytC5656'],
                "userId" : "ghfdgYT67"
                
            }
